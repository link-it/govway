Modello canonical interno per il gateway LLM (GovLLM).

Il canonical è il pivot tra dialetti front-door (OpenAI Chat Completions, Anthropic Messages) e formati back-door dei provider. È modellato sul dialetto più espressivo (Anthropic Messages, block-based) e include campi opzionali per le feature specifiche di OpenAI.

Caratteristiche:

  • Block-based content: ogni messaggio porta una lista di blocchi tipizzati (text, tool_use, tool_result). In futuro: image, document, thinking.
  • Round-trip lossless sulla coppia stesso-dialetto: OpenAI → canonical → OpenAI ≡ OpenAI.
  • Indipendente dal dialetto di ingresso: il campo sourceDialect di CanonicalChatRequest è solo audit/debug e non guida la logica di trasformazione.
